# 🧠 AI · ScienceTech 뉴스 브리핑 앱 — 요구사항 정리

## 1️⃣ 핵심 목표 (Core Vision)
"AI + 과학기술 최신 정보를 자동 수집 → 요약 → 정리 → 필터링해서 한국어 중심으로 빠르게 소비 가능한 브리핑 앱"
단순 RSS 리더가 아니라:
- 한국어 요약 제공
- 핵심 포인트 추출
- 중복 제거
- 클러스터링
- 정확한 개별 기사 원문 링크 제공
- 로컬 + 정적 Pages 동작 지원

## 2️⃣ 현재 시스템 구조
### 백엔드 (FastAPI + SQLite)
- RSS 수집
- URL canonicalize (utm 제거)
- 본문 추출
- 중복 제거 (URL + 제목 유사도)
- 요약/번역
- DB 저장

### 정적 파이프라인
- RSS → generate_data.py → feed.json + articles/*.json
- GitHub Pages fallback

### 프론트엔드 (Next.js)
- NewsList
- ArticleDetail
- 정적 fallback 감지
- 원문 링크 유효성 체크

## 3️⃣ 주요 기능 요구사항
### 1) 기사 수집 관련
- OpenAI 말고 다양한 소스 추가
- AI + ScienceTech 전체 영역 확대
- 오래전 글도 수집 가능
- 최대 기사 수 확장 (150 → 더 확장 가능)
- 중복 제거 강화
- 클러스터링
원하는 방향: 단일 블로그 앱 ❌ / 통합 브리핑 앱 ✅

### 2) 원문 링크 정확성
- “블로그 메인 주소가 아니라, 정확한 개별 기사 페이지 링크”
- 홈/섹션 URL은 무효 처리
- 유효 URL만 버튼 노출
- 없으면 “원문 링크 없음” 표시
향후: RSS alternate 우선, GUID fallback, 링크 검증 고도화

### 3) 요약 시스템
현재: rule-based summarize_and_translate()
향후: LLM 기반 요약, 한국어 자연스러운 재구성, 키워드 태깅, 산업/학계 구분

### 4) 필터 시스템
현재: 토픽(AI/SciTech/All), 기간(24h/7d/30d), 검색, 정렬(최신/오래된)
원하는 UX: 기간 너무 빡세면 자동 완화, 0개면 fallback 표시, 정렬 안정성

### 5) 동작 모드
- 로컬 모드: FastAPI 연결, 수집 실행 버튼 활성
- GitHub Pages 모드: 정적 데이터만 사용, 수집 버튼 비활성, fallback 배지 표시
이 구조는 반드시 유지

## 4️⃣ 앱 성격
AI/과학기술 “전문가용 브리핑 대시보드”
대상: 엔지니어/연구자/테크 종사자/투자 관심자/AI 트렌드 모니터링

## 5️⃣ 다음 단계 확장 방향
- 트렌드 분석 탭: 최근 7일 키워드, 소스 비율, AI vs SciTech 비중
- 클러스터 상세 보기: 같은 이슈 묶기, “이 이슈 관련 기사 5개”
- 개인화: 관심 태그 저장, 읽음 처리, 북마크
- 알림: RSS 1시간 수집, 특정 키워드 알림
